{% trans "Gitcoin supports 3 flexible ways to set up a project to ensure the best results." %}



Traditional Project


One Worker Contributes One Worker is Paid

Good For

Small projects that require ramp up time, or a way to reserve a
project for a specific contributor.

What to Know

A traditional project can be set up for contributors to be able
to start work immediately without approval or can be set up
as approval required.

Example Traditional Project Flow


Cooperative Project


Multiple Contributors Multiple Contributors Paid

Good For

Large projects (broken up into parts), augmenting team skillsets.

What to Know

Cooperative bounties may or may not require approval to work.

When working on a cooperative bounty, if you are the first person
explain your approach given that others may join you.

If you are joining after the first contributor, check to see how
the project has progressed and where you can jump in to contribute.

The amount paid for a cooperative project is at the discretion of
the funder.

Example Cooperative Project Flow


Contest Project


Multiple Contributors One Contributor Paid*

Good For

Sourcing ideas, creative, searching for the best ways to solve a problem, bug hunt.

What to Know

Contest bounties may or may not require approval to work.

The funder reserves the right to pay only one or more than
one contributor for the contest at their discretion. This depends
on the type of contest listed and the number of winners alloted.
